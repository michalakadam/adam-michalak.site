Programmer is a craftsman who combines commands into programs that can be interpreted by a computer and executed repeatedly. Every program is an attempt to delegate execution of tasks that could otherwise be done manually. Delegation requires up-front investement of time and effort. However, it pays off in spades. In this article I would like to motivate you to bring your programming skills to the next level and start delegating repetitive shell tasks with scripts. I am not going to give you any hows. Once you are convinced you want to acquire a powerful skill of scripting, check this excellent Bash 101 tutorial. For now let's focus on the reasons for writing and using shell scripts.
Save time with each run

Investigate the way you use command line and identify sequences of instructions that you use repeatedly. All of those that are longer than one command are worth turning into a script. I know that at first it does not seem like a good investment. However, the more proficient you will get at scripting, the bigger return will come with every script. Let's take a look at one of my most recent creations to illustrate that. This website is served from a remote server without use of any web hosting services. This means that I have to take care about deployment process. Here are the steps that need to be performed to deploy each change to a hosting server:

  • Local change is pushed to a remote GitHub repository.
  • Remote server is informed about it.
  • Change is pulled from a remote repo to the server.
  • Website is built on the server with Angular CLI.
  • Compiled files are copied to a folder with website files.
  • Application hosting website (nginx) is restarted.

Whole operation takes 3-4 minutes and is composed of about 15 commands. It is certainly enough to keep me from deploying to production frequently which is not very agile... Therefore I spent several hours on writing script that does it for me before I wrote a single line of code for this website. Did it pay off? Hell yeah! I deploy to production regularly every time I create a new feature - up to several times a day.

Err no more

Shell scripts are a great way to store complex sequence of commands. For example there is a lot of boiler plate files that have to be added when starting a new Java project. Readme.md, pom.xml, .gitignore not to mention custom git hooks and folder structure with main class. It's lot of typing, copying, pasting. This sequence has to be remembered and rerun everytime. Probability of making an error by mistyping or forgeting something is quite high.

Initial Java project structure is quite verbose.

Every step of project's structure creation can be performed from the command line. For example generation of folder structure that follow Maven standard directory layout can be performed with just one command. Put all those instructions in a sequence, save it into a file with .sh extension, make it executable and you are all set. Use across unix systems

Great thing about scripts is that they are platform agnostic. They can be executed on any Unix-based system like Linux and MacOS but also Windows (wtf, did I just said that?!) thanks to the recent WSL feature. This makes script writing a one-time effort. Once you spend time creating a script, it will serve you forever. You can take it to the next level: create a script that downloads your scripts from cloud storage of any sorts and installs all the necessary executables after a launch of a brand new operating system.
